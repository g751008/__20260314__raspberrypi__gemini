## 概述

`test.py` 是一個 **Open WebUI Filter（過濾器）插件**的範例程式。

Open WebUI 的 Filter 機制讓開發者可以在對話流程的兩個關鍵時間點插入自訂邏輯：
- **inlet**：使用者訊息送往 AI 模型「之前」
- **outlet**：AI 模型回應「之後」、顯示給使用者「之前」

---

## 程式結構

```
Filter
├── Valves（管理員設定）
├── UserValves（使用者設定）
├── __init__()（初始化）
├── inlet()（入口攔截器）
└── outlet()（出口攔截器）
```

---

## 各部分說明

### Valves（管理員設定）

```python
class Valves(BaseModel):
    priority: int = 0       # 過濾器執行優先順序
    max_turns: int = 8      # 管理員設定的對話輪數上限
```

- 由 WebUI 管理員在後台設定
- 套用到所有使用者
- `priority`：當有多個 Filter 時，決定執行順序（數字越小越先執行）
- `max_turns`：整個系統允許的最大對話輪數

### UserValves（使用者設定）

```python
class UserValves(BaseModel):
    max_turns: int = 4      # 使用者自己的對話輪數上限
```

- 每位使用者可在自己的設定頁面調整
- 只影響自己的對話
- 預設值（4）比管理員上限（8）更嚴格

### `__init__()` 初始化

```python
def __init__(self):
    self.valves = self.Valves()
```

- 建立 `Valves` 設定物件，讓管理員設定值生效
- 若需要自訂檔案處理，可啟用 `self.file_handler = True`

### `inlet()` 入口攔截器

**執行時機**：使用者送出訊息後，AI 模型收到之前

**主要功能**：對話輪數限制

```
實際上限 = min(使用者設定上限, 管理員設定上限)
```

例如：
| 使用者設定 | 管理員設定 | 實際上限 |
|-----------|-----------|---------|
| 4         | 8         | 4       |
| 10        | 8         | 8       |
| 3         | 8         | 3       |

若超過上限，會拋出例外（Exception），中止本次請求並回傳錯誤訊息。

### `outlet()` 出口攔截器

**執行時機**：AI 模型回應後，顯示給使用者之前

**目前功能**：僅印出除錯資訊，直接回傳原始回應（未做修改）

可在此加入：回應內容過濾、敏感詞遮蔽、日誌記錄等功能。

---

## 運作流程

```
使用者輸入訊息
      ↓
  inlet() 攔截
  - 檢查對話輪數
  - 超過上限 → 拋出例外，中止
  - 未超過   → 繼續
      ↓
  AI 模型處理
      ↓
  outlet() 攔截
  - 可修改或記錄回應
      ↓
  顯示給使用者
```

## 延伸應用

可以在 `inlet()` 加入：
- 關鍵字過濾（拒絕特定詞彙的請求）
- 使用者權限驗證
- 請求內容修改或補充

可以在 `outlet()` 加入：
- 回應內容審查
- 敏感資訊遮蔽
- 使用量統計記錄
